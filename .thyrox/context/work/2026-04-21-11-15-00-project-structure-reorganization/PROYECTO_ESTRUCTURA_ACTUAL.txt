CONVENCIONES DEL PROYECTO
================================================================================

DOCUMENTACIÓN:
  - Sin INDEX.md (se vuelve obsoleto)
  - Sin emojis ni iconos
  - Sin README.md en carpetas de código
  - Frontmatter YAML en documentos principales

CÓDIGO POWERSHELL:
  - Estructura modular (Functions/Public, Functions/Private)
  - Convenciones: Verb-Noun (Get-, Invoke-, Write-)
  - Validación robusta de input
  - Logging completo
  - Idempotencia garantizada

COMMITS:
  - Convencionales: feat(scope): description
  - Scopes: functions, validation, logging, docs, tests, chore
  - Ejemplos:
    feat(functions): agregar Invoke-InputValidation
    fix(validation): corregir validación de idioma
    docs(requirements): documentar UC-001
    test(scenarios): agregar tests de idempotencia

WORK PACKAGES:
  - Nomenclatura: YYYY-MM-DD-HH-MM-SS-nombre
  - Ubicación: .thyrox/context/work/
  - Ejemplo: 2026-04-21-10-30-00-uc-documentation

STAGES (Simplificados de THYROX):
  1. DISCOVER      - Análisis, definición del problema
  2. DESIGN/SPEC   - Diseño, especificación de requisitos
  3. PLAN          - Planificación de implementación
  4. IMPLEMENT     - Implementación en código
  5. TEST/VALIDATE - Tests, validación, QA
  6. FINALIZE      - Documentación, release

DIFERENCIAS vs MSOI
================================================================================

  - Sin validación robusta de input
  - Sin verificación de integridad

  - Logging limitado

  - Estructura monolítica
